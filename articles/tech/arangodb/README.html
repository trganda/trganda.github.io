        • arangodb

        arangodb 是一款开源的可扩展的图形数据库系统。

        安装

        从 Download 下载，解压后执行如下命令启动数据库

        arangodb start

        之后可通过 Web 页面 http://localhost:8529/ 进行访问，也可通过控制台访问。默认包含 _system 数据库和名为 root 用户。

        可通过如下命令重设 root 用户密码

        sudo systemctl stop arangodb3 # 关闭服务器
arango-secure-installation

        数据结构

        arangodb 将数据分为 3 层

        database->collections->documents

        Documents

        其中 documents 中的数据以 json 形式进行存储，每一对 k-v 被称为 attribute，所有的 document 都包含两个默认属性，_id 和 _key，可以手动指定，若没有则会自动生成，它们都是唯一的，创建后无法修改。此外还有 _rev 属性，由系统进行维护，无需进行设定。

        每个 document 都有与之关联的 key，也就是 _key 属性，它的值需要遵循如下规范：

        • 由大小写字母，阿拉伯数字，_、 -、 .、 @、 (、 )、 +、 ,、 =、 ;、 $、 !、 *、 '、 % 组成。
        • 最大长度 256 个字节，大小写敏感。

        Collections

        collections 是 documents 的集合，它包含两种类型，document collection/vertex collections（作为图的节点时） 和 edge collections，前者为默认类型。

        若是类型为 edge collections 『边』，则需要包含 _from 和 _to 属性，它们分别指向 vertex collections 中 document 的 _id。从 _from 到 _to 表示一条有向边。

        collections 的名称需要遵循以下规范：

        • 由大小写字母，阿拉伯数字，_ 和 - 组成，用户定义的 collections 必需以字母开头，系统定义的 collections 必需以 _ 开头。
        • 最大长度 256 个字节，大小写敏感。

        当使用 --database.extended-names 参数启动 arangodb 时可支持拓展命名功能『实验性』，相比前者不同的是：

        • 支持大多数 UTF-8 能够表示的字符，例如中文，表情等，但不能包含控制字符，/ 和 :。
        • 首位不能包含空格或制表符。
