    Github Action 是 Github 提供的免费的 CICD 功能，对于开源仓库的使用限制很小。Action 是一个 workflow，它可由一个或多个 job 组成。

    Workflow 配置文件语法

    workflow 的配置文件使用的 yam/yaml 格式，可从 Learn yaml in Y Minutes 快速了解该格式的相关细节和语法规则。下面是一个 workflow 配置文件，描述了大部分常用字段和含义：

    变量

    在 action 允许定义并使用变量，可以全局环境中定义

    也可以在某个 step 中定义，POSTGRES_HOST 和 POSTGRES_PORT 只对当前执行的命令（脚本） node client.js 有效。

    默认环境变量

    在 workflow 的运行环境中包含默认的环境变量，下面列举常用的环境变量，完整列表见 default environment variables。

    VariableDescription
    GITHUB_ACTION_REPOSITORY当前 step 所运行的 action 所属的仓库，如使用了第三方 action，actions/checkout，则返回 actions/checkout
    GITHUB_API_URLgithub api 地址，https://api.github.com
    GITHUB_REPOSITORY仓库所有者和存储库名称
    GITHUB_REPOSITORY_OWNER仓库所有者的名字
    GITHUB_SERVER_URLgithub 地址，https://github.com
    RUNNER_ARCHjob 所在平台的架构
    RUNNER_OSjob 所在平台的操作系统
    RUNNER_NAME当前 job 的名字
    RUNNER_TEMP临时目录，action 运行结束后删除

    表达式执行

    workflow 中可通过语法

    ${{ <expression> }}

    来执行表达式，表达式的执行结果需要是为以下类型之一

    • boolean
      • true 或 false
    • null
      • null
    • number
      • JSON 支持的任何数字格式。
    • string
      • ${{ }} 中使用字符串时需要将字符串用 ' 进行包裹。如果字符串中要包含 '，请用 '' 进行转义来表示。

    函数

    workflow 中内置了部分 Functions，可查看文档进行参考。

    Context

    workflow 中包含了许多 context 对象，可在表达式中引用：

    • github：当前 workflow 运行时环境对相关信息，见 github context。
    • env：当前 workflow 中『workflow、job、step』设置的变量，见 env context。
    • vars：在仓库或组织的 enviroment 中配置的变量, 见 vars context。
    • secrets：在仓库或组织的 enviroment 中配置的 secret 变量，见 secrets context.
    • needs：包含定义为当前 job 的直接依赖项的所有其它 jobs 的输出，见 needs context。
    • 更多 context 对象见 context。

    Events that Trigger Workflow

    触发 workflow 的方式主要分为，自动、定时和手动『我自己分的』。定时触发的放松，可以使用 cron 的语法进行设置，可参考 Github - Actions 定时任务。手动触发可通过在 on 字段中设定 workflow_dispatch，并在 Action 页面中启用相关 workflow 即可。

    自动触发的 events 有很多，具体可参看 Events that trigger workflows - GitHub Docs。通过 event 触发可以指定单个或多个 events，

    Action Marketplace

    在构建 action 时可优先从 action marketplace，寻找所需的 action，如果没有满足条件的再自行编写。使用方式很简单，在 steps 字段下，使用 use 引入你所需使用的第三发 action

    某些 action 需要一些参数，如 actions-js/push@master 可将 action 对当前仓库的修改内容 commit 并推送回仓库中，但需要使用的环境变量 GITHUB_TOKEN。

    Github Artifacts

    如果相在当前 workflow 的不同 job 之间共享数据，可以将数据存储到文件中，并上传至 Github 中作为一个 artifacts。这需要使用到 actions/upload-artifact，例如当前 job 将结果保持至 output.log 文件中，并上传至 github ，命名为 output-log-file。

    调试

    action 默认输出的日志内容是比较有限的，如果想了解更详细的运行过程，可通过开启调试模式在 action 的运行过程中输出更详细的日志内容。要开启调试模式，需要在当前仓库中设置环境变量 ACTIONS_RUNNER_DEBUG 为 true 『推荐在 secret 中设置』。
