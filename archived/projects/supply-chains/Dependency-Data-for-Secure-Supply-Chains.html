Dependency Data for Secure Supply Chains

Jun 12, 2023, 12 min read

  • #deps
  • #open-source-security

谷歌的 Open Source Security 团队开放了开源软件的数据查询接口 deps.dev。目前支持的生态包括 GO, NPM, CARGO, MAVEN, PYPI, NUGE，其数据从公开范围收集，具体说明如下：

  • 对于 Maven 包，除了 Maven Central 之外，它还从 Jenkins 和 Google 注册中心收集工件（一般为 Jar 包）。
  • 对于 PyPI，它只收集作为 wheels 和 sdists 发布的包。
  • 对于 Go，它只收集通过 proxy.golang.org 获取的模块，以及那些被这些模块声明为依赖项的模块。
  • 对于托管在 GitHub、GitLab 和 Bitbucket 上的项目，仅收集与服务已知的包关联的项目。

API 使用说明 §

具有以下接口：

GetPackage §

GetPackage 返回有关包的信息，包括其可用版本的列表，默认版本标记为已知。

GET /v3alpha/systems/{packageKey.system}/packages/{packageKey.name}

参数

  • packageKey.system: string
    • 包所属的生态
  • packageKey.name: string
    • 包名

示例，/v3alpha/systems/npm/packages/%40colors%2Fcolors

{
  "packageKey": {
    "system": "NPM",
    "name": "@colors/colors"
  },
  "versions": [
    {
      "versionKey": {
        "system": "NPM",
        "name": "@colors/colors",
        "version": "1.4.0"
      },
      "isDefault": false
    },
    {
      "versionKey": {
        "system": "NPM",
        "name": "@colors/colors",
        "version": "1.5.0"
      },
      "isDefault": true
    }
  ]
}

响应的结构如下，其中 versions[].isDefault 表示是否为默认版本，即 latest 版本或默认安装的版本。

@startjson
#highlight "packageKey" / "system"
#highlight "versions" / "0" / "versionKey" / "name"
#highlight "versions" / "0" / "versionKey" / "version"
{
	"packageKey": {
	   "system": "NPM",
	   "name": "@colors/colors"
	},
	"versions": [
		{
	      "versionKey": {
	        "system": "NPM",
	        "name": "@colors/colors",
	        "version": "1.4.0"
	      },
	      "isDefault": false
	    }
	]
}
@endjson

GetVersion §

GET /v3alpha/systems/{versionKey.system}/packages/{versionKey.name}/versions/{versionKey.version}

  • versionKey.system: string
    • 包所属的生态
  • versionKey.name: string

  • versionKey.version: string

GetDependencies §

GetDependencies 返回给定包版本的解析依赖关系图。

GET /v3alpha/systems/{versionKey.system}/packages/{versionKey.name}/versions/{versionKey.version}:dependencies

参数

  • versionKey.system: string
    • 包所属的生态
  • versionKey.name: string
    • 包名
  • versionKey.version: string
    • 包的版本

示例，18.2.0:dependencies

{
  "nodes": [
    {
      "versionKey": {
        "system": "NPM",
        "name": "react",
        "version": "18.2.0"
      },
      "bundled": false,
      "errors": []
    },
    {
      "versionKey": {
        "system": "NPM",
        "name": "js-tokens",
        "version": "4.0.0"
      },
      "bundled": false,
      "errors": []
    },
    {
      "versionKey": {
        "system": "NPM",
        "name": "loose-envify",
        "version": "1.4.0"
      },
      "bundled": false,
      "errors": []
    }
  ],
  "edges": [
    {
      "fromNode": 0,
      "toNode": 2,
      "requirement": "^1.1.0"
    },
    {
      "fromNode": 2,
      "toNode": 1,
      "requirement": "^3.0.0 || ^4.0.0"
    }
  ],
  "error": ""
}

响应结构如下

@startjson
#highlight "packageKey" / "system"
#highlight "versions" / "0" / "versionKey" / "name"
#highlight "versions" / "0" / "versionKey" / "version"
{
	"nodes": [
		{
		    "versionKey": {
	        "system": "NPM",
	        "name": "react",
	        "version": "18.2.0"
		},

	    },
		{
			"versionKey": {
				"system": "NPM",
				"name": "js-tokens",
				"version": "4.0.0"
			},
			"bundled": false,
			"errors": []
		},
		{
			"versionKey": {
				"system": "NPM",
				"name": "loose-envify",
				"version": "1.4.0"
			},
			"bundled": false,
			"errors": []
		}
	],
	"edges": [
		{
			"fromNode": 0,
			"toNode": 2,
			"requirement": "^1.1.0"
		},
		{
			"fromNode": 2,
			"toNode": 1,
			"requirement": "^3.0.0 || ^4.0.0"
		}
	],
	"error": ""
}
@endjson

响应字段说明

  • nodes[]: object[]
    • 依赖图的节点，第一个节点是图的根。
  • nodes[].versionKey
    • 包的信息，生态，包名，版本。
  • nodes[].bundled: boolean
    • 如果为真，此为捆绑依赖项，对于捆绑依赖项，verisonKey 的 name 会描述其构成。如，name 为 a>1.2.3>b>c，表示版本为 1.2.3 的包 a 的依赖关系图的一部分，并且具有一个 local 名称 c ，可能与 global 名称为 c 的包相同或者不同。
  • nodes[].errors[]: string[]
    • 与图的此节点关联的错误，例如未解决的依赖性要求，没有固定格式。
  • edges[]: object[]
    • 依赖图的边
  • edges[].fromNode: number
    • 声明依赖关系的起始节点，指定了在 node 种的索引。
  • edges[].toNode: number
    • 声明依赖关系的目标节点，指定了在 node 种的索引。
  • edges[].requirement: string
    • 声明 fromNode 节点的依赖需求，它的值在不同生态中可能不一样，例如，在 npm 中，^1.0.0 表示 fromNode 节点需要至少 1.0.0 版本的 toNode 作为依赖。

GetAdvisory §

GetAdvisory 返回有关 OSV 托管的安全公告的信息

GET /v3alpha/advisories/{advisoryKey.id}

参数

  • advisoryKey.id: string
    • OSV 标识符

示例，GHSA-2qrg-x229-3v8q

{
    "advisoryKey": {
        "id": "GHSA-2qrg-x229-3v8q"
    },
    "url": "https://osv.dev/vulnerability/GHSA-2qrg-x229-3v8q",
    "title": "Deserialization of Untrusted Data in Log4j",
    "aliases": [
        "CVE-2019-17571"
    ],
    "cvss3Score": 9.8,
    "cvss3Vector": "CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H"
}

  • title: string
    • 描述。
  • aliases[]: string[]
    • 别名，如 CVE 编号。
  • cvss3Score: number
    • CVSS v3 分数，在 [0,10] 范围内，分数越高表示严重程度越高。
  • cvss3Vector: string
    • CVSS v3 矢量字符串，描述在不同维度上的严重性。

Query §

查询有关多个包版本的信息，这些信息可以通过名称、包内容哈希或两者指定。

GET /v3alpha/query

参数

  • hash.type: string
    • 哈希算法，MD5、SHA1、SHA256、SHA512 之一
  • hash.value: string
    • 哈希值。
  • versionKey.system: string
    • 包所属的生态
  • versionKey.name: string
    • 包名
  • versionKey.version: string
    • 包的版本

示例，query?hash.type=SHA1&hash.value=ulXBPXrC%2fUTfnMgHRFVxmjPzdbk%3d
