        CycloneDX

        May 11, 20238 min read

        • cyclonedx
        • bom

        CyclonedX 是由 OWASP 推出的 SBOM 规范，并提供了一系列工具来辅助开源项目构建自己的 SBOM 文件，支持多种格式，包括 xml、json、protobuf。在官方仓库中提供了该格式在不同方面的应用 示例，

        下图为 CycloneDX 的抽象模型。

        CycloneDX 的设计宣称，除了基本的 SBOM 能力，还具备其他 BOM 功能。它们分别是：

        • CycloneDX - Software-as-a-Service BOM (SaaSBOM)
          • 支持云原生应用程序的清单服务、端点以及数据流和分类，由 services 部分提供支持。
        • CycloneDX - Vulnerability Exploitability eXchange (VEX)
          • 支持在使用它们的软件的上下文中传达漏洞信息，如某个依赖组件存在的漏洞，漏洞是否会影响当前软件等。
        • CycloneDX - Hardware Bill of Materials (HBOM)
          • 支持硬件物料清单，可列举物联网、ICS 和其他类型的嵌入式和连接设备的库存硬件组件。
        • CycloneDX - Operations Bill of Materials (OBOM)
          • 支持运行时环境、配置和附加依赖项的全栈清单，可存放于 metadata.component.properties，见示例 Standalone 或示例 Decoupled，OBOM 由 BSIMM 标准引入。
        • CycloneDX - Vulnerability Disclosure Report (VDR)
          • 传递影响组件和服务的已知和未知漏洞。
        • CycloneDX BOM-Link
          • 从其他系统或其他 BOM 中引用组件、服务或 BOM 中的漏洞。
        • CycloneDX - Bill of Vulnerabilities
          • 在系统和漏洞情报来源之间共享漏洞数据。

        Package URL(PURL)

        Package URL 是一类标准，用于统一表示开源软件在开源软件宇宙中的位置。例如使用了什么语言，在哪个仓库平台，软件名称，版本等信息，它的形式和 URL 很类似，在 2018 年的 FOSDEM 会议上被提出。

        各组成部分的定义如下：

        • scheme：固定为 pkg，为未来的拓展做保留，必须
        • type：包的类型或者来源，如 maven、npm、nuget、gem、pypi 等，必须
        • namespace：名称空间，例如 Maven 中的 groupid，Docker 镜像中的所有者，Github 上的用户名或组织名，可选
        • name：包的名称，必须
        • version：版本号，可选
        • qualifiers：额外的元数据，例如操作系统，架构平台等，可选
        • subpath：相对于 namespace 的子路径，可选

        Common Platform Enumeration (CPE)

        CPE 是信息技术系统、软件和包的结构化命名方案。基于统一资源标识符 (URI) 的通用语法，CPE 包括正式名称格式、根据系统检查名称的方法以及将文本和测试绑定到名称的描述格式。

        以下是 CPE 产品词典的当前 官方版本。该词典提供了一份商定的官方 CPE 名称列表。该词典以 XML 格式提供，可供公众使用。请经常回来查看，因为 CPE 产品词典将继续增长以包括所有过去、现在和未来的产品版本。当修改或添加新名称时，CPE 词典每晚更新一次。

        https://nvd.nist.gov/products/cpe

        自 2009 年 12 月起，美国国家漏洞数据库正在接受对官方 CPE 词典的投稿。有兴趣提交 CPE 名称的组织应通过 cpe_dictionary@nist.gov 联系 NVD CPE 团队，以寻求处理其提交的帮助。

        由 NIST 托管和维护的 CPE 词典可由非政府组织自愿使用，在美国不受版权保护。但是，NIST 会赞赏署名。

        CPE 目前最新 标准 版本为 2.3，例如对于 internet_explorer ，使用 well-formed CPE name (WFN) 方法

        wfn:[part="a",vendor="microsoft",product="internet_explorer",
version="8\.0\.6001",update="beta"]

        它的 CPE URL Binding（兼容 2.2）格式如下

        cpe:/a:microsoft:internet_explorer:8.0.6001:beta

        当携带 Formatted String Binding（2.3 新增）时，其格式如下

        cpe:2.3:a:microsoft:internet_explorer:8.0.6001:beta:*:*:*:*:*:*

        并提供了正则表达式用于检查，见 xsd:anyURI 和 xsd:pattern
