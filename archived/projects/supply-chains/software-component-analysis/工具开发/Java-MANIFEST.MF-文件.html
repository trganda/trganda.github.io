        Java MANIFEST.MF 文件

        Jun 12, 20234 min read

        • manifest
        • jar

        MANIFEST.MF 是 Java 中包的描述文件，用于记录包的元信息。

        结构

        默认的 MANIFEST

        在 openjdk11 下，执行如下命令

        生成的 Jar 包中的 MANIFEST 文件内容如下

        Manifest-Version: 1.0
Created-By: 11.0.3 (AdoptOpenJDK)

        自定义 MANIFEST

        MANIFEST 允许自定义内容，增加字段值和描述。

        例如，有 MANIFEST 文件 manifest.txt，把它添加进 Jar 包中

        生成的 MANIFEST 文件内容如下

        Manifest-Version: 1.0
Built-By: baeldung
Created-By: 11.0.3 (AdoptOpenJDK)

        Headers

        Header 字段由 key 和 value 组成，形式如下

        key: value

        Main Header，也就是默认生成的字段，通常提供了基础信息，如：

        • Manifest-Version: the version of the specification
        • Created-By: the tool version and vendor that created the manifest file
        • Multi-Release: if true, then this is a Multi-Release Jar
        • Built-By: this custom header gives the name of the user that created the manifest file

        Entry Point 和 Classpath

        可以在 MANIFEST 中指定 Jar 包的入口类以及 Classpath，从而避免运行 Jar 包时，手动指定。

        • Main-Class: the package and name of the class with a main method (no .class extension)
        • Class-Path: a space separated list of relative paths to libraries or resources，一般用的很少。 示例如下：
        Main-Class: com.baeldung.Application
Class-Path: core.jar lib/ properties/

        Package 版本和封装

        以下是标准头部

        • Name: the package
        • Implementation-Build-Date: the build date for the implementation
        • Implementation-Title: the title of the implementation
        • Implementation-Vendor: the vendor for the implementation
        • Implementation-Version: the implementation version
        • Specification-Title: the title for the specification
        • Specification-Vendor: the vendor for the specification
        • Specification-Version: the specification version
        • Sealed: 如果为 true，则表示所有类文件都位于此 Jar 包中，默认为 false 如，中 MySQL Connector 驱动的 Jar 包中有如下内容，描述了实现的 JDBC 标准的版本
        Specification-Title: JDBC
Specification-Version: 4.2
Specification-Vendor: Oracle Corporation
Implementation-Title: MySQL Connector/J
Implementation-Version: 8.0.16
Implementation-Vendor: Oracle

        OSGI

        OSGI bundle 中的 Jar 包的 MANIFEST 会包含如下头部：

        • Bundle-Name: title
        • Bundle-SymbolicName: a unique identifier
        • Bundle-Version: version
        • Import-Package: packages and versions the bundle depends on
        • Export-Package: bundle packages and versions available for use

        Section

        MANIFEST 文件中的 Section 分为两种，Main 和 Per-Entry。在 Main 部分的内容是每个 Jar 文件都可能出现的，而 Per-Entry 中的内容只会出现在指定的包或类名下。

        Per-Entry 中的内容优先级高于 Main 中的内容，如果出现同名字段，Per-Entry 中的内容会被采用。Per-Entry 通常存放版本和数字签名信息。见下面的示例：

        # Main Section
Implementation-Title: baeldung-examples
Implementation-Version: 1.0.1
Implementation-Vendor: Baeldung
Sealed: true

# Pre-Entry Section
Name: com/baeldung/utils/
Sealed: false

        其中，Pre-Entry Section 表示用于描述 com/baeldung/utils/ 包。

        总结

        MANIFEST.MF 中可表示包名称的字段

        • Name
        • Bundle-Name
        • Short-Name
        • Extension-Name
        • Implementation-Title

        MANIFEST.MF 中可表示包版本的字段

        • Implementation-Version
        • Specification-Version
        • Plugin-Version
        • Bundle-Version

        MANIFEST.MF 中可表示包 LICENSE 的字段

        • Bundle-License
        • Plugin-License-Name
