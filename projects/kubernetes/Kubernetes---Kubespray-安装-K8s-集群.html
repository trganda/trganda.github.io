Kubespray 安装 Kubernetes 集群

Oct 11, 2023, 5 min read

初始化 §

首先克隆 kubespray 至本地，之后安装 ansible，官方文档推荐使用 python 的虚拟环境进行安装。主要执行以下命令前前不要切换至克隆的 kubespray 目录

VENVDIR=kubespray-venv
KUBESPRAYDIR=kubespray
python3 -m venv $VENVDIR
source $VENVDIR/bin/activate
cd $KUBESPRAYDIR
pip install -U -r requirements.txt

执行完成后会进行虚拟 shell 环境。

(kubespray-venv) trganda@hostname:~/kubespray$

修改配置文件 §

拷贝配置文件

cp -rfp inventory/sample inventory/mycluster

之后修改 inventory/mycluster 目录下的配置文件 hosts.yaml 和 inventory.ini，你也可以通过 kubespray 提供的脚步初始化这两个文件

declare -a IPS=(10.10.1.3 10.10.1.4 10.10.1.5)
CONFIG_FILE=inventory/mycluster/hosts.yaml python3 contrib/inventory_builder/inventory.py ${IPS[@]}

注意替换对应的 ip 地址为自己的。

检查 hosts.yaml 或 inventory.ini 文件，确保信息准确。

hosts.yaml §

ansible 支持多种格式，这里选择使用 yaml。

hosts.yaml 文件中存储的是主机相关的信息，例如，IP 地址，用户名，私钥等。kubespray 会使用 ansible 根据 hosts.yaml 中的信息通过 ssh 连接至对应的主机。

这里需要注意的是 ansible_host 和 ip 有一定的区别，ansible_host 使用 ssh 连接的地址，而 ip 是安装后 kubernetes 各个服务绑定的地址。所以如果你使用的是云主机，或弹性 IP，那么它的外部 IP 和内部 IP 地址是不一样的；此时若想从外部使用进行安装，则需要修改 ansible_host 为外部 IP。

这里是直接在云主机内部环境中进行安装的。如果从外部安装，将 ansible_host 修改成公网地址，或者添加，并添加 access_ip

all:
  hosts:
    node1:
      ansible_host: x.x.x.x
      ansible_user: ubuntu
      ip: 10.0.0.5
      access_ip: x.x.x.x
      etcd_access_address: x.x.x.x
    node2:
      ansible_host: x.x.x.x
      ansible_user: ubuntu
      ip: 10.0.0.84
      access_ip: x.x.x.x
    node3:
      ansible_host: x.x.x.x
      ansible_user: ubuntu
      ip: 10.0.0.242
      access_ip: x.x.x.x
    node4:
      ansible_host: x.x.x.x
      ansible_user: ubuntu
      ip: 10.0.0.92
      access_ip: x.x.x.x
  children:
    kube_control_plane:
      hosts:
        node1:
    kube_node:
      hosts:
        node1:
        node2:
        node3:
        node4:
    etcd:
      hosts:
        node1:
    k8s_cluster:
      children:
        kube_control_plane:
        kube_node:
    calico_rr:
      hosts: {}

根据官方文档，默认情况不需要修改如下内容，除非你知道自己在做什么。

k8s_cluster:
      children:
        kube_control_plane:
        kube_node:
    calico_rr:
      hosts: {}

通常只需要更改 kube_control_plane、kube_node 和 etcd 中的节点就好。上面的配置文件中，指定 node1 作为 kube_control_plane；node1、node2、node3 作为 nodes；node1 作为 etcd 数据库的存储位置，当然你可以同时指定多个

etcd:
  hosts:
	node1:
	node2:
	node3:

配置 §

除了前面两个最基本的配置文件，还有很多其它的配置文件，毕竟 kuebernetes 相关的组件很多，并且还包括插件，如果需要进行定制则根据自身需要检查并修改这些文件。

例如想使用 kubectl top 命令的功能，需要安装 metrics server，此时可在 inventory/mycluster/group_vars/k8s_cluster/addons.yml 文件中修改为如下配置

metrics_server_enabled: true

如果想开启 dashboard 或 helm，也可在 addons.yml 中进行配置。

安装 §

确认好配置文件后，还有一件事需要做，你需要为各个主机指定同一个公钥『这样做更方便，只需使用一个私钥』，当然你也可以在 hosts.yaml 文件中为每一台机器指定私钥文件位置。

首先创建一对密钥 key

ssh-keygen -t ed25519 -b 4096 -C "xxx@mail.com"

之后将生成的公钥『id_ed25519.pub』copy 至远端主机，默认写入文件 ~/.ssh/authorized_keys 中。

ssh-copy-id -i ~/.ssh/id_ed25519.pub -f user@hostname

-f 表示强制模式，如果 authorized_keys 中有相同的 key 则直接覆盖。

对各个主机执行上述操作后，ansible 就可以使用同一个私钥连接至各个主机了。之后使用如下命令进行安装

ansible-playbook -i inventory/mycluster/hosts.yaml -b -v --private-key=~/.ssh/id_ed25519 cluster.yml

如果各个机器之间的架构不同，例如混合着 x86、arm64 则会遇到问题，可参考 Kubernetes-Kubespray 多架构下的安装 进行解决。

如果是跨区域的安装，例如混杂着云主机和 VPS，可以参考 Kubernetes-Kubespray 跨区域安装。
