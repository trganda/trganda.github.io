Oracle Cloud 提供一下称为 Always Free 的服务，只要注册即可享受下图中提供的云服务。

有关这项服务的具体介绍，见 Always Free。需要注意的是此项服务有资源回收策略，见如下说明

Idle Always Free compute instances may be reclaimed by Oracle. Oracle will deem virtual machine and bare metal compute instances as idle if, during a 7-day period, the following are true:

  • CPU utilization for the 95th percentile is less than 15%
  • Network utilization is less than 15%
  • Memory utilization is less than 15% (applies to A1 shapes only)

这里的 95th 是表示统计计算方法，见下图

云主机的端口访问问题 §

如果在云主机内部监听某项端口以提供服务，为了让外部能够访问该服务，需要配置 VCN 子网的安全策略并在云主机中配置防火墙策略。

这里以操作系统为 ubuntu 为例进行说明，可修改 /etc/iptables/rules.v4 文件来放行入站端口，先备份

cp /etc/iptables/rules.v4 /etc/iptables/rules.v4.bak

假设要允许外部对 8080 端口的访问，则在其中添加如下内容

-A INPUT -p tcp --dport 8080 -j ACCEPT

之后重新加载防火墙策略

iptables-restore < /etc/iptables/rules.v4

有关 iptables 的具体内容，见 iptables 学习记录。

完成上述步骤后需要在安全策略中同步配置，放行 8080 端口。
