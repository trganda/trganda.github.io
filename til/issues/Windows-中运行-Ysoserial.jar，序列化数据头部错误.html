Windows 中运行 Ysoserial.jar，序列化数据头部错误

Aug 18, 2023, 5 min read

  • #win
  • #ysoserial
  • #powershell

起因 §

在 Windows 中使用 ysoserial.jar 生成 payload 并写入文件

java -jar ysoserial.jar CommonsBeanutils1 "notepad" > poc.ser

文件 poc.ser 的头部却不是 ACED

尝试解决 §

查看 ysoserial 与输出相关的源码

public class GeneratePayload {
   private static final int INTERNAL_ERROR_CODE = 70;
   private static final int USAGE_CODE = 64;
 
   public static void main(final String[] args) {
      if (args.length != 2) {
         printUsage();
         System.exit(USAGE_CODE);
      }
      final String payloadType = args[0];
      final String command = args[1];
 
      final Class<? extends ObjectPayload> payloadClass = Utils.getPayloadClass(payloadType);
      if (payloadClass == null) {
         System.err.println("Invalid payload type '" + payloadType + "'");
         printUsage();
         System.exit(USAGE_CODE);
         return; // make null analysis happy
      }
 
      try {
         final ObjectPayload payload = payloadClass.newInstance();
         final Object object = payload.getObject(command);  // [1]
         PrintStream out = System.out;
         Serializer.serialize(object, out);
         ObjectPayload.Utils.releasePayload(payload, object);

默认输出对象为 System.out。尝试将序列化结果写入文件，在 [1] 后添加如下代码

ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("pocs.ser"));
oos.writeObject(object);

输出文件的头部是正常的。

从前后的结果对比可以发现另一个最大的区别，前者字符后跟着 00，猜测它占用了 2 个字节。那么这个问题应该和控制台或系统的默认编码方式有关。

我尝试了以下几种方式

java "-Dfile.encoding=UTF-8" -jar ysoserial.jar CommonsBeanutils1 "notepad" > poc.ser

或者修改控制台输出编码

chcp 65001

并没有什么用。

我想可能是重定向符对编码做了什么，于是尝试

java "-Dfile.encoding=UTF-8" -jar ysoserial.jar CommonsBeanutils1 "notepad" | Out-File poc.ser -Encoding utf8

输出结果正常了一点点，头部依然有问题，为什么会变成 ef bb bf ee 84 88

后记 §

我还是不清楚在使用 powershell 执行这段命令的时候，发生了什么导致结果是非预期的。

更新 2023.2.28 §

我再次尝试 powershell，不过这次换了编码，得到了预期的结果。

java -jar ysoserial.jar CommonsBeanutils1 "notepad" | Out-File poc.ser -Encoding oem

其中 ome 表示使用 MS-DOS 和控制台程序的默认编码。
