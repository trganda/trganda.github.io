Arch Linux Installtation

Sep 21, 2023, 5 min read

  • #arch
  • #installation

Pre-Request §

Download the iso file of archlinux from offical site.

And boot it from your (virtual) machine.

Preparation §

Tips: If you are installing arch on a virtual machine, after arch connect to network, you can change the password or live system by

passwd

and use ssh to remote control the installation.

Connect to Network §

If you install archlinux from virtual machine, the connection should be automatic connected.

To set up a network connection in the live enviroment, go through the following steps:

Update The Systeml Clock §

Use timedatectl to ensure the system clock is accurate

timedatectl set-ntp true

Partition The Disks §

You can use lsblk or fdisk tool to modified your block device, it’s recommand to use fdisk, run

fdisk -l

to inspire your devices.

Typically, you can partition your disks like below with GPT

Mount pointPartitionPartition typeSuggested size
/mnt/boot1/dev/efi_system_partitionEFI system partitionAt least 300 MiB
[SWAP]/dev/swap_partitionLinux swapMore than 512 MiB
/mnt/dev/root_partitionLinux x86-64 root (/)Remainder of the device

So, you can run the command below (suppose your disk device is /dev/sda),

fdisk /dev/sda

to entire the interactive model and

  • create a GPT
  • create a partition /dev/sda1 with type efi system
  • create a swap partition /dev/sda2
  • create a main partition /dev/sda3 for mount root path

Format The Partitions §

For efi system partition, format with

mkfs.fat -F 32 /dev/sda1

:warning: Only format the EFI system partition if you created it during the partitioning step. If there already was an EFI system partition on disk beforehand, reformatting it can destroy the boot loaders of other installed operating systems.

Initialize the swap space

mkswap /dev/sda2

and format the /dev/sda3 to a normal Ext4 filesystem.

mkfs.ext4 /dev/sda3

Mount the File Systems §

Create a folder named boot.

mkdir -p /mnt/boot

Mount the root volume to /mnt.

mount /dev/sda3 /mnt

Mount the EFI system partition

mount /dev/sda1 /mnt/boot

Enable swap space with swapon(8):

swapon /dev/sda2

For the next step, genfstab(8) will detect mounted file systems and swap space.

Installation §

Before the installation, it’s recommand to chose the best mirror for you region. On the live system, after connecting to the internet, reflector updates the mirror list by choosing 20 most recently synchronized HTTPS mirrors and sorting them by download rate. Or, you can run

reflector -c {Your Country} -a 10 --sort rate --save /etc/pacman.d/mirrorlist

Install Essential Packages §

Use the pacstrap(8) script to install the base package, Linux kernel and firmware for common hardware

pacstrap /mnt base linux linux-firmware vim sudo

Tips: You can substitute linux for a kernel package of your choice. You could omit the installation of the firmware package when installing in a virtual machine or container.

Configure The System §

Fstab §

Generate a fstab file (use -U or -L to define by UUID or labels, respectively):

genfstab -U /mnt >> /mnt/etc/fstab

Check the resulting /mnt/etc/fstab file, and edit it in case of errors.

Chroot §

Change root into the new system:

arch-chroot /mnt

Time Zone §

Set the time zone

ln -sf /usr/share/zoneinfo/Region/City /etc/localtime

and run hwclock(8) to generate /etc/adjtime

hwclock --systohc

This command assumes the hardware clock is set to UTC. See System time standard for details.

Localization §

Edit /etc/locale.gen and uncomment en_US.UTF-8 UTF-8 and other needed locales. Generate the locales by running:

locale-gen

Create the locale.conf(5) (by touch /etc/locale.conf) file, and set the LANG variable (include the other needed locales you enabled with locale-gen) accordingly:

# /etc/locale.conf
LANG=en_US.UTF-8

Network Configuration §

It’s recomand to installl NetworkManager package for manager you network,

pacman -S networkmanager

You can create a connection with nmtui tool provided by networkmanager package and enable the service startup with

systemctl enable NetworkManager

Set Root Password §

Set the [passwod](Root password) of root

passwd

And create a not-root user

useradd -m -g users -s /bin/bash {user_name}

and set the password for this user

passwd {user_name}

Add the privilege for use sudo command

vim /etc/sudoers
# add the following line after root ALL=(ALL) ALL
# {user_name} ALL=(ALL) ALL

Tips: Make sure you have installed the sudo package.

Install Boot Loader §

Choose and install a Linux-capable boot loader. I have use Grub here, install the package first

pacman -S dosfstools grub efibootmgr

And run

grub-install --target=x86_64-efi --efi-directory=/boot --recheck

Generate the configuration file

grub-mkconfig -o /boot/grub/grub.cfg

Reboot §

Exit the chroot environment by typing exit or pressing Ctrl+d.

Optionally manually unmount all the partitions with umount -R /mnt, this allows noticing any “busy” partitions, and finding the cause with fuser(1).

Finally, restart the machine by typing reboot, any partitions still mounted will be automatically unmounted by systemd. Remember to remove the installation medium and then login into the new system with the root account.aura

Now, you can reboot your system, and start using your arch linux :grinning:.

Post Installation §

Yay §

yay support downloading packages from the Arch User Repository (AUR), which is a community-maintained repository providing thousands of third-party packages in the form of installation scripts, also known as PKGBUILDs.

To install yay on your Arch desktop, first, download the following dependencies:

sudo pacman -S --needed base-devel git

Then, clone the yay repository using the git clone command:

git clone https://aur.archlinux.org/yay.git

Change your present working directory to the newly-downloaded yay folder using the cd command:

cd yay

Finally, use the makepkg command to build and install yay:

makepkg -si
